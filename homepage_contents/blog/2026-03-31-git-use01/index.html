【備忘録】Gitの使い方-01_gitへのアップロード

2026/03/31 0 PV 0 いいね

本Webサイト2個目のブログになります！
今回は私の備忘録的なものになるのですが、Gitの使い方を書きます。
ソフトウェアを開発するうえでバージョン管理ツールは必須ですが、1人で開発する際は、何も考えずにただただ上げていただけだったので、これを機に学習しようと思います！

Gitへのアップロード概要

gitでアップロードするための構成は以下のようになっている

ワークツリー→ステージ→ローカルリポジトリ→リモートリポジトリ

ワークツリーは現在作業している場所
ステージはコミットの前段階の場所
ローカルリポジトリはコミットの履歴をローカルで保存している場所
リモートリポジトリはコミットの履歴をリモートで保存している場所

リポジトリを作成する

①Gitリポジトリを作成する

git init

git管理したいフォルダで上記コマンドを実行することで新しいGitリポジトリを作成することができる。
フォルダ内に.gitが作られる。

②ステージングにアップロードする

git add <ファイル名>

上記コマンドを実行することで、対象を指定してステージングにアップロードすることができる。
"git add ."だとすべてになる。

③ステージングを確認する

git status

上記コマンドを実行すことでステージングにあるファイルを確認することができる。

④commitする

git commit -m "コミットメッセージ"

上記コマンドを実行することでローカルリポジトリにアップロードを行うことができる。

⑤リモートリポジトリにアップロードする

git remote add <好きな名前> <リモートリポジトリURL>
git branch -M main
git push -u origin main

上記コマンドを実行することでリモートリポジトリにアップロードする
<好きな名前>は保存先に付与する名前。慣例的にはoriginを付与するよう
githubの保存先を登録して、
ブランチ名mainを登録し、そこに移動し、
Githubにアップロードしているというような形。
