## Gitへのアップロード概要
gitでアップロードするための構成は以下のようになっている
```
ワークツリー→ステージ→ローカルリポジトリ→リモートリポジトリ
```
ワークツリーは現在作業している場所<br>
ステージはコミットの前段階の場所<br>
ローカルリポジトリはコミットの履歴をローカルで保存している場所<br>
リモートリポジトリはコミットの履歴をリモートで保存している場所<br>

## リポジトリを作成する
### ①Gitリポジトリを作成する
```
git init
```
git管理したいフォルダで上記コマンドを実行することで新しいGitリポジトリを作成することができる。<br>
フォルダ内に.gitが作られる。

### ②ステージングにアップロードする
```
git add <ファイル名>
```
上記コマンドを実行することで、対象を指定してステージングにアップロードすることができる。<br>
"git add ."だとすべてになる。

### ③ステージングを確認する
```
git status
```
上記コマンドを実行すことでステージングにあるファイルを確認することができる。

### ④commitする
```
git commit -m "コミットメッセージ"
```
上記コマンドを実行することでローカルリポジトリにアップロードを行うことができる。

### ⑤リモートリポジトリにアップロードする
```
git remote add <好きな名前> <リモートリポジトリURL>
git branch -M main
git push -u origin main
```
上記コマンドを実行することでリモートリポジトリにアップロードする<br>
<好きな名前>は保存先に付与する名前。慣例的にはoriginを付与するよう<br>
githubの保存先を登録して、<br>
ブランチ名mainを登録し、そこに移動し、<br>
Githubにアップロードしているというような形。
